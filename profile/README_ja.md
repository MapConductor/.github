# Map Conductor - クラウド連携型地図SDK 統合プラットフォーム -

[![License: MIT](https://img.shields.io/badge/License-Apache2-yellow.svg)](https://opensource.org/licenses/Apache2)
[![PRs Welcome](https://img.shields.io/badge/PRs-welcome-brightgreen.svg?style=flat-square)](http://makeapullrequest.com)

**地図アプリ開発を、もっと自由に、すべての開発者へ。**

Map Conductor (マップコンダクター) は、乱立する地図SDKの差異を吸収し、開発の複雑さを解消するクラウド連携型の地図SDK統合プラットフォームです。まるでコンテナオーケストレーションにおけるKubernetesのように、MapConductorは地図アプリケーション開発の「標準」となることを目指します。

---

## 概要

地図を使ったアプリケーション開発は、多くの「あるある」な悩みに満ちています。

*   AndroidとiOSで微妙に異なる挙動
*   利用する地図SDK（Google Maps, Mapbox, ArcGISなど）によって全く違うコードの書き方
*   一度選ぶと後戻りできない「ベンダーロックイン」
*   複雑なライセンス体系

これらの課題に対応するうち、本来注力すべき「地図の上で、いかに情報を魅力的に見せるか」という核心部分から、開発者はどんどん離れていってしまいます。

**MapConductorは、この「本末転倒」を解決します。**

共通のAPIを通じて、背景でどの地図SDKが動いているかを意識することなく、あなたのアプリケーションを構築できます。これにより、開発コストを劇的に削減し、リソースが限られた組織でも、誰もが自由に地図を活用できる「地図開発の民主化」を実現します。

## 主な特徴

*   **マルチSDK対応:**
    *   単一のコードで、Google Maps, Mapbox, Here, ArcGISなど、複数の主要な地図SDKを操作できます。
    *   将来的には50以上のSDKへの対応を目指しています。

*   **共通API:**
    *   MapConductorが提供する統一されたメソッドを使うだけで、地図の表示、操作、データプロットが可能です。
    *   SDKごとの細かい「お作法」を学ぶ必要はもうありません。

*   **ベンダーロックインからの解放:**
    *   プロジェクトの要件に合わせて、最小限のコード修正でバックエンドの地図サービスを自由に乗り換えられます。

*   **ハイブリッドアーキテクチャ:**
    *   **ミドルウェアSDK (OSS):** 基本的な地図表示機能やSDK接続アダプターは、オープンソースとして無償で提供します。
    *   **商用クラウドサービス:** アセットトラッキング、ジオフェンス、Webベースの管理画面など、より高度でビジネスユースに耐えうる機能を有償で提供します。

*   **高性能レンダリング:**
    *   独自の描画技術により、数万件を超える大量のデータを地図上にプロットしても、スムーズな拡大・縮小・移動を実現することを目標にしています。
